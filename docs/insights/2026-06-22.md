**Proposed new marts/metrics**

1. **Corridor‑Delay Mart** – a dimensional table keyed by *corridor_id* (e.g., “McClellan Hwy”, “Bridge St”) with attributes: `median_delay_min`, `otp_pct`, `stop_count`, `peak_delay_hour`. Enables rapid “hot‑spot” dashboards and supports root‑cause analyses (signal timing, lane restrictions).

2. **Early‑Late Balance Index (ELBI)** – a fact table storing, per route‑stop‑hour, `early_minutes`, `late_minutes`, `otp_pct`. Compute a normalized ELBI = (late – early) / (late + early + 1). A negative ELBI flags over‑scheduled segments (early‑heavy), a high positive ELBI flags chronic lateness. Can be sliced by route, stop, or time‑of‑day.

3. **Stop‑Level Dwell‑Variance Metric** – capture for each stop the `median_dwell_time`, `stddev_dwell`, and `delay_attributable_to_dwell` (difference between observed stop arrival and scheduled arrival minus upstream travel delay). This isolates whether the +10 min at Bridge St stops stems from prolonged boardings/alightings versus travel‑time variability, guiding targeted interventions (e.g., all‑door boarding, stop consolidation).
